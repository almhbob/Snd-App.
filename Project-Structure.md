# هيكلية مشروع Snd — Project Structure

> "مشروع Snd يعتمد على هيكلية بيانات مركزية (Google Sheets) مرتبطة بواجهة Glide. يتم التحكم في ظهور مقدمي الخدمة عبر فلتر إداري صارم (Admin Verification Layer) لضمان مصداقية كل 'ولدنا' يظهر في التطبيق."

---

## 1. قاعدة البيانات — Snd Database (Google Sheets)

الملف: **Snd Database**  
الصفحة: **Main_Data**

| العمود | الوصف |
|---|---|
| الاسم الكامل | اسم مقدم الخدمة |
| رقم الهاتف | رقم التواصل المباشر |
| التخصص | نوع الخدمة المقدمة |
| صورة الهوية (للرصد الإداري) | رابط صورة الهوية للمراجعة الداخلية |
| حالة التوثيق (المفتاح السري) | القيم: `موثوق` / `قيد الانتظار` |
| التقييم | تقييم المستخدمين من 1 إلى 5 |
| إيميل المستخدم (للتعرف عليك كآدمن) | البريد الإلكتروني للمشرف |

---

## 2. الواجهة — Glide App (Snd)

- المنصة: [Glide Apps](https://www.glideapps.com/)
- المصدر: Google Sheets → Snd Database
- اسم المشروع: **Snd**

---

## 3. بروتوكول التوثيق — Admin Verification Layer

يعمل النظام وفق الآلية التالية:

1. يتم تسجيل كل مقدم خدمة جديد بحالة **"قيد الانتظار"** تلقائياً.
2. يراجع المشرف (Admin) البيانات وصورة الهوية.
3. عند الموافقة، يتم تغيير **حالة التوثيق** إلى **"موثوق"**.
4. يظهر مقدم الخدمة للجمهور **فقط** بعد تغيير الحالة.

### شرط الفلتر في Glide:
```
العمود: حالة التوثيق
الشرط: Is
القيمة: موثوق
```

```
  └─────────────────────────────────┘
```
